Public Endpoints (No Authentication Required)
GET /v1/health No Auth
Health check endpoint to verify if the API server is running.
curl -X GET http://localhost:3000/v1/health
Response:
{ "status": "OK", "timestamp": "2025-01-01T00:00:00.000Z", "uptime": "0 seconds" }
POST /v1/auth/admin/login No Auth
Admin user login endpoint. Returns JWT access and refresh tokens.
curl -X POST http://localhost:3000/v1/auth/admin/login \ -H "Content-Type: application/json" \ -d '{"email":"admin@example.com","password":"password"}'
Response:
{ "access_token": "eyJhbGciOiJIUzI1NiIs...", "refresh_token": "eyJhbGciOiJIUzI1NiIs...", "user": { "id": "user-id", "email": "admin@example.com", "role": "admin" } }
GET /v1/users/public No Auth
Get publicly available user information (filtered data).
curl -X GET http://localhost:3000/v1/users/public
GET /v1/public/* No Auth
Access to public data and resources.
GET /v1/doctors No Auth
Get list of available doctors (public information).
curl -X GET http://localhost:3000/v1/doctors
Protected Endpoints (Authentication Required)

Note: All endpoints below require JWT authentication. Include the access token in the Authorization header: Authorization: Bearer YOUR_ACCESS_TOKEN

GET /v1/dashboard/api Auth Required
Get dashboard statistics and data for admin users.
curl -X GET http://localhost:3000/v1/dashboard/api \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
GET /v1/users Auth Required
Get all users with optional filtering by role.
curl -X GET "http://localhost:3000/v1/users?role=patient" \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
POST /v1/users Auth Required
Create a new user account.
GET /v1/users/{id} Auth Required
Get specific user details by ID.
curl -X GET http://localhost:3000/v1/users/USER_ID \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
GET /v1/appointments Auth Required
Get all appointments with optional filtering.
curl -X GET "http://localhost:3000/v1/appointments?status=scheduled&date=2025-01-01" \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
POST /v1/appointments Auth Required
Create a new appointment.
GET /v1/availability Auth Required
Get doctor availability schedules.
POST /v1/availability Auth Required
Set or update doctor availability.
GET /v1/invoices Auth Required
Get invoices with optional filtering.
GET /v1/notifications Auth Required
Get user notifications.
POST /v1/notifications Auth Required
Send a notification.
GET /v1/chat/messages Auth Required
Get chat messages between users.
POST /v1/chat/messages Auth Required
Send a chat message.
POST /v1/video/call Auth Required
Initiate a video call session.
GET /v1/analytics/* Auth Required
Access various analytics data and reports.
GET /v1/master-services Auth Required
Get master services configuration.
GET /v1/settings Auth Required
Get application settings.
PUT /v1/settings Auth Required
Update application settings.
Authentication Guide

Getting Access Token

1. Use the login endpoint to get your access token:

POST /v1/auth/admin/login Content-Type: application/json { "email": "your-email@example.com", "password": "your-password" }

2. Include the token in all protected API requests:

Authorization: Bearer YOUR_ACCESS_TOKEN_HERE

Token Expiration

Access tokens expire after a certain period. Use the refresh token to get a new access token when needed.

Common Response Codes

400 - Bad Request
401 - Unauthorized
404 - Not Found
500 - Internal Server Error
